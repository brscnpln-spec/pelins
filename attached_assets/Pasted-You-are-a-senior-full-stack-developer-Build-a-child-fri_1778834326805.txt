You are a senior full-stack developer. Build a child-friendly “Daily Dinosaur Duel” feature for an existing Raspberry Pi touchscreen home dashboard.

Context:
- The dashboard already shows calendar and Munich weather.
- There is empty space below the weather area.
- Add a new widget/component in that lower empty area.
- Target user is a child named Kerem.
- No AI integration should be required.
- The duel must change automatically once per day.
- The UI must work well on a Raspberry Pi touchscreen.

Feature name:
Daily Dinosaur Duel

Main flow:
1. Show today’s duel in the lower dashboard area.
   Example:
   “Today’s Dino Duel”
   “T. rex vs Diplodocus”
2. Both dinosaur names must be large, clear, and tappable.
3. Kerem taps one dinosaur as his guess.
4. After tapping, show a short duel animation for 3–5 seconds.
5. If the selected dinosaur is the correct winner:
   - Show positive child-friendly feedback.
   - Example: “Correct! T. rex wins this duel! 🦖”
6. If the selected dinosaur is wrong:
   - Show gentle, cute negative feedback.
   - Example: “Good try! Diplodocus is huge, but today T. rex had the advantage. 🦕”
7. After the guess, lock the duel until the next calendar day.
8. Until the next day, show the result state:
   - Today’s duel
   - Kerem’s selected dinosaur
   - Correct winner
   - Short explanation
   - “New duel tomorrow”

Important rules:
- Do not use OpenAI or any external AI API.
- Use a local predefined duel list.
- The daily duel should be selected deterministically based on the current date.
- The same day must always show the same duel.
- Store the user’s daily answer in localStorage.
- If the page refreshes, the result must remain visible.
- At midnight/local date change, a new duel becomes available.
- The widget must be offline-friendly.
- No backend is required unless the existing app already has one.

Data model:
Create a local array of duel objects like this:

{
  id: "trex-vs-diplodocus",
  dinoA: "T. rex",
  dinoB: "Diplodocus",
  winner: "T. rex",
  explanationCorrect: "Correct! T. rex was a powerful predator with a massive bite.",
  explanationWrong: "Good try! Diplodocus was enormous, but T. rex had the stronger attack in this duel.",
  funFact: "T. rex had one of the strongest bites of any land animal."
}

Add at least 30 duels so the feature can rotate for a month.

Use child-friendly but reasonably realistic outcomes. Avoid scary, violent, bloody, or aggressive wording. Use words like:
- advantage
- stronger defense
- faster
- bigger
- clever hunter
- protected by armor
- powerful bite

Do not use:
- killed
- blood
- destroyed
- ripped apart
- violent descriptions

Daily selection logic:
- Get today’s local date as YYYY-MM-DD.
- Convert the date into a stable number.
- Use modulo with the duel list length.
- Example logic:
  const todayKey = new Date().toLocaleDateString("en-CA");
  const dayNumber = todayKey.replaceAll("-", "");
  const index = Number(dayNumber) % DUELS.length;
  const todayDuel = DUELS[index];

Storage logic:
- localStorage key should include today’s date:
  dino-duel-result-YYYY-MM-DD
- Store:
  {
    duelId,
    selected,
    winner,
    isCorrect,
    answeredAt
  }
